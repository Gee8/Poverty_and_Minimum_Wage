Project Overview

The intention of this project is to evaluate how major economic conditions affect poverty rates. This project analyzes if and how major economic factors such as population, education spending, welfare spending, crime rate, unemployment rate, divorce rate, homeownership rate, effective minimum wage, average wage index, consumer price index (CPI), and inflation rate can predict poverty rates in the United States.

Our projections were developed using data from the Bureau of Labor Statistics, Federal Reserve Bank, Social Security Administration, Center on Poverty and Social Policy at Columbia, Federal Bureau of Investigation crime data, Centers for Disease Control and Prevention, Inflation Data and Kaggle. We projected poverty rates at the state and national levels using machine learning algorithms. Multiple linear regression and decision tree regressor techniques were used to model the relationship between the multiple economic indicators and poverty level, and forecast the poverty rate based on these factors. Linear regression models were selected because we are using multiple independent variables as inputs to predict a continuous dependent variable or outcome.

Data sourced from numerous CSV files was read into Jupyter Notebook and Google Colaboratory and transformed using Python and Pandas, then loaded into Amazon RDS, SQLAlchmey and a PostgresSQL database on the cloud. A Flask application was used to display the project analysis and data visualizations built using Tableau, JavaScript, HTML and CSS on webpages.





Questions We Attempted to Answer

1) Can poverty rates be extrapolated from historical economic indicators?

2) Which economic indicators have the highest correlation with poverty rates?

3) How does adjusting certain economic indicators affect the poverty rate?

Visualizations

The results of this project are displayed in a website. The webpages include the analysis of the results, interactive and static graphs, interactive tables, and an interactive poverty predictor. For the interactive graphs, the user will be able to select the state(s) and feature(s) they wish to see a historical line plot of. An example of the static graphs is a heatmap of a correlation matrix for the economic features, displaying how strong, or weak, of a relationship there is with each of the other features. For the poverty predictor, the user will pick a state, which will select the saved machine learning model for that state. They will then be able to adjust the values of the features, and these customized features will be fed into the trained machine learning model to predict the poverty rate for the users customized features.
